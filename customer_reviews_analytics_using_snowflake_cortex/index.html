In this Quickstart guide, you will help the fictitious food truck company, Tasty Bytes, identify where their customer experience may be falling short at the truck and business level by leveraging Snowflake Cortex within Snowflake Notebook.

They collect customer reviews from multiple sources and languages to better understand improvement areas and boost customer satisfaction and loyalty.

Prerequisites

  • Familiarity with Python
  • Familiarity with the DataFrame API
  • Familiarity with Snowflake
  • Familiarity with Snowpark

What You'll Need

  • A Snowflake account in a region where Cortex is supported
    • Cortex functions used: Complete, Translate, Sentiment
    • Models used: snowflake-arctic, mistral-large
  • Snowflake Notebook access (currently private preview)

What You'll Learn

  • Translating multilingual customer reviews
  • Categorizing unstructured review text
  • Deriving customer sentiment
  • Identifying review issues
  • Generating LLM-assisted actions

What You'll Build

Use Cortex Translate to conditionally translate non-English reviews into English for easier analysis.

Use Cortex Complete to:

  • Get ratings from reviews using one-shot learning
  • Predict recommendation likelihood using zero-shot learning

Use Cortex Sentiment to calculate a sentiment score (-1 to 1) for each review.

Use Cortex Complete with a one-shot example to extract category-wise sentiment (e.g., food, price, service).

Use Cortex Complete to:

  • Aggregate and analyze top 100 negative reviews to summarize business-level issues
  • Identify trucks with lowest average sentiment and summarize issues

Use Cortex Complete to generate an email summarizing issues and recommending fixes to a truck franchise owner.

Congratulations! You've successfully enabled customer review analytics using Snowflake Cortex within a Snowflake Notebook.
